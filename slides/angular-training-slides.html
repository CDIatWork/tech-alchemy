Signals — Reaktiver State

export class TaskListComponent {
  // Signal erzeugen (wie eine reaktive Variable)
  tasks = signal<Task[]>([]);
  filter = signal('all');

  // Computed: automatisch abgeleiteter Wert
  activeTasks = computed(() =>
    this.tasks().filter(t => !t.completed)
  );

  // Wert ändern
  addTask(task: Task) {
    this.tasks.update(list => [...list, task]);
  }
}
  • signal<T>(initial) — erzeugt einen reaktiven Wert
  • signal() — liest den aktuellen Wert (Aufruf als Funktion!)
  • signal.set() / signal.update() — ändert den Wert
  • computed() — automatisch abgeleiteter Wert, wie ein CDI Producer mit @Observes

Kernunterschied: CDI ist typbasiert (Klasse + Qualifier). Angular ist token-basiert (Klasse oder InjectionToken) mit hierarchischen Injektoren.

Services erstellen

@Injectable({ providedIn: 'root' })
export class TaskService {
  private http = inject(HttpClient);
  private apiUrl = '/api/tasks';

  findAll(): Observable<Task[]> {
    return this.http.get<Task[]>(this.apiUrl);
  }

  findById(id: string): Observable<Task> {
    return this.http.get<Task>(`${this.apiUrl}/${id}`);
  }

  create(task: Partial<Task>): Observable<Task> {
    return this.http.post<Task>(this.apiUrl, task);
  }
}
  • @Injectable({providedIn: 'root'}) = @ApplicationScoped
  • inject(HttpClient) = @Inject HttpClient
  • Rückgabe: Observable<T> (asynchron!)

Observables verstehen

Ein Observable<T> ist ein asynchroner Datenstrom — wie ein CompletableFuture<T>, aber mächtiger.

// Service aufrufen und Ergebnis verarbeiten:
this.taskService.findAll().subscribe({
  next: (tasks) => this.tasks.set(tasks),
  error: (err) => this.error.set('Fehler beim Laden'),
});

// Mit Operatoren (pipe):
this.taskService.findAll().pipe(
  catchError(err => {
    this.error.set('Fehler: ' + err.message);
    return of([]);  // Fallback: leere Liste
  }),
  finalize(() => this.loading.set(false))
).subscribe(tasks => this.tasks.set(tasks));
  • .subscribe() startet den Request (wie CompletableFuture.thenAccept())
  • pipe() ermöglicht Transformationen: map(), filter(), catchError()

Loading-State & Fehlerbehandlung

export class TaskListComponent implements OnInit {
  tasks = signal<Task[]>([]);
  loading = signal(true);
  error = signal<string | null>(null);

Buch: Kapitel 11.6 — Sicherheit im Frontend

AuthService — JWT-Verwaltung

@Injectable({ providedIn: 'root' })
export class AuthService {
  private http = inject(HttpClient);
  currentUser = signal<User | null>(null);
  isLoggedIn = computed(() => this.currentUser() !== null);

  login(email: string, password: string) {
    return this.http.post<{ token: string }>(
      '/api/auth/login', { email, password }
    ).pipe(
      tap(res => {
        localStorage.setItem('token', res.token);
        this.currentUser.set(this.decodeToken(res.token));
      })
    );
  }

  logout() {
    localStorage.removeItem('token');
    this.currentUser.set(null);
  }

  getToken(): string | null {
    return localStorage.getItem('token');
  }
}

Route Guards

// auth.guard.ts
export const authGuard: CanActivateFn = (route, state) => {
  const auth = inject(AuthService);
  const router = inject(Router);

  if (auth.isLoggedIn()) {
    return true;
  }
  // Weiterleiten zum Login mit Rückkehr-URL
  return router.createUrlTree(
    ['/login'], { queryParams: { returnUrl: state.url } }
  );
};

// In den Routes verwenden:
{ path: 'tasks',
  canActivate: [authGuard],
  loadComponent: () => import('...') }
  • CanActivateFn — funktionaler Guard (moderner Ansatz)
  • Gibt true (erlaubt) oder UrlTree (Redirect) zurück

HTTP-Interceptor — JWT automatisch anhängen

// auth.interceptor.ts
export const authInterceptor: HttpInterceptorFn =
    (req, next) => {
  const auth = inject(AuthService);
  const token = auth.getToken();

  if (token) {
    const cloned = req.clone({
      setHeaders: {
        Authorization: `Bearer ${token}`
      }
    });
    return next(cloned);
  }
  return next(req);
};

// In app.config.ts registrieren:
providers: [
  provideHttpClient(withInterceptors([authInterceptor]))
]
Parallele: Ein HTTP-Interceptor in Angular entspricht einem Servlet-Filter oder CDI-Interceptor — er umschließt jeden ausgehenden Request.

JSF → Angular: Migration Cheat Sheet

View-Schicht

  • XHTML-Seiten → Components
  • #{} → {{ }}
  • ui:repeat → @for
  • rendered → @if
  • h:form → Reactive Forms
  • h:commandButton → (click)

Backend-Anbindung

  • @Named Bean → @Injectable Service
  • Form-Submit → REST + HttpClient
  • Session-State → Signals + localStorage
  • Servlet Filter → HTTP-Interceptor
  • ViewConfig → Routes-Array
  • @Secured → canActivate Guard

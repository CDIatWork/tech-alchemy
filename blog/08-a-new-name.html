AI Journey · Blog Post #8

[CDI@Work]:AI – A New Name for What Comes Next

February 2026

Until now, every AI contribution to this project was additive. Claude redesigned the layout. Translated the book. Wrote three new chapters. Built a test suite. But the original eight chapters — Gerhard Petracek’s core text about CDI and Apache DeltaSpike — remained untouched. The AI worked around the original, never into it.

That’s about to change. The project now carries a new name to reflect it.

CDI@Work
↓
[CDI@Work]:AI

Why the Name Changed

The original name, CDI@Work, belonged to a specific book by a specific author. It described CDI 1.0 through early 2.0, all in the context of Java EE 6 and Apache DeltaSpike. That content was written by a human, and it stood on its own for years.

The new name signals a shift in scope. The :AI suffix makes explicit that this is no longer the original text alone — it’s the original text plus AI-authored improvements, updates, and extensions. The brackets around the original name preserve attribution: CDI@Work is still the foundation. The AI builds on top of it.

Concretely, this means the original Chapters 1–8 are now candidates for improvement. Not rewriting — the teaching approach and structure stay. But clarifications, updated examples, corrected terminology, better explanations where the original was terse. The same kind of editorial refinement that Post #7 described for Chapter 11, but now applied to the book’s foundation.

What Changed in the Rename

The rename touched every file in the project — 38 occurrences across 13 files:

  • Both book editions — page titles, meta descriptions, and the sidebar heading that every reader sees first
  • All seven blog posts — titles, meta tags, navigation links, and footers
  • Blog index — page title, header tag, card link text, and footer
  • Project documentation — README, CLAUDE.md, and the test suite README

A regex verification confirmed zero remaining standalone occurrences of the old name. Every instance now reads [CDI@Work]:AI.

The Story So Far

Looking back at the eight blog posts, a clear arc emerges:

  • Post #1: Redesign — new layout, same content
  • Post #2: New chapter — Jakarta EE bridge
  • Post #3: Visual before-and-after
  • Post #4: Test suite — guarding structural integrity
  • Post #5: New chapter — Quarkus migration
  • Post #6: New chapter — Angular frontend
  • Post #7: Improving a chapter — bridging the knowledge gap
  • Post #8: Renaming the project — AI collaboration goes deeper

Posts 1–6 were about adding things alongside the original. Post 7 was the first time AI improved content it had already written. Post 8 opens the door to improving content the original author wrote. Each step asks a little more of the collaboration.

The principle: The original work is the foundation, not a museum piece. Preserving an author’s intent doesn’t mean freezing their text. It means understanding what they were teaching and finding clearer ways to teach it — with full transparency about what changed and who changed it.

What Comes Next

With the rename in place, the next posts in this series will cover actual improvements to the original chapters. The disclaimer boxes that mark Chapters 9–11 as AI-authored will extend to any original chapter that receives significant changes. The test suite will continue to verify structural integrity across both editions.

The name change is small. What it represents is not.

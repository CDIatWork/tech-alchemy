AI Journey · Blog Post #9

Polishing the Original

February 2026

Post #8 announced that the original chapters were now candidates for improvement. This post covers what happened next: all eight original chapters — the entire foundation of the book — received a systematic prose polish. No paragraphs were removed. No code was changed. The goal was simple: make every chapter read a little more smoothly, fix accumulated grammar issues, and add forward-looking hints where the technology has moved on.

The Rules

The editorial pass followed strict rules, agreed with the project maintainer before the first edit:

  • Start with German — the German edition is the original language of the book, so edits begin there
  • Keep every paragraph — reword for fluency, never drop content
  • Preserve tone and attitude — the author’s teaching style stays intact
  • Never touch code — code snippets and technical terms remain exactly as written
  • Fix spelling and grammar — even in paragraphs that read fine otherwise
  • Mark co-author additions — any new content about modern alternatives is clearly attributed

Disclaimers and Co-Author Hints

Each of the eight chapters now begins with a yellow disclaimer box:

Hinweis: Der Text dieses Kapitels wurde von Claude (AI built by Anthropic) als Co-Autor überarbeitet. Diese Überarbeitung erfolgte ohne nachträgliches Review durch den Originalautor. Verwendung auf eigene Verantwortung.

This is the same pattern used for the AI-authored Chapters 9–11, making it immediately visible which chapters have been touched. The English edition carries an equivalent disclaimer.

At the end of each chapter, a green co-author hint points readers to related modern developments covered later in the book. These hints bridge the gap between the original Java EE 6/7 content and the Jakarta EE world of Chapters 9–11.

What Didn’t Change

The structural integrity of the book is completely preserved. Not a single code listing was modified. No sections were added, removed, or reordered. The tip boxes, chapter boundaries, and anchor links are all identical to before.

By the numbers: The book still contains 11 chapters, 88 sections, 350 code listings, and 105 tip/warning boxes — verified by 198 automated tests across both the German and English editions after every single commit. All green, every time.

The eight commits follow a clean progression: one per chapter, each touching only book.html and book_en.html. The git history makes it straightforward to review exactly what changed in each chapter and to revert any individual change if needed.

The Bigger Picture

This editorial pass is what the rename to [CDI@Work]:AI was about. The AI is no longer just adding new material alongside the original — it’s improving the original itself, with clear rules, full transparency, and automated verification.

The book that started as a LaTeX-generated German website about CDI 1.0 now reads more fluently in its original language, carries forward-references to three new chapters about Jakarta EE, Quarkus, and Angular, and is continuously verified by a test suite that didn’t exist six weeks ago. Every change is attributed. Every commit is atomic. Every test is green.

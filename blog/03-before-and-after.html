From LaTeX to Modern Web – A Visual Before & After

February 2026

When a friend helped me turn [CDI@Work]:AI from LaTeX into a website years ago, it was a big step forward — suddenly the book was accessible as an online resource instead of just a PDF. A frameset layout with purple navigation, clean structure, and everything a reader needed. It served its purpose well for a long time. But web expectations have evolved since then, and the design started to show its age.

Here's what the transformation looks like side by side:

What Changed

Everything about the presentation — nothing about the content. The original German text and all 292 code listings were preserved exactly as written. What changed was how it's all presented.

Navigation Old: flat frameset with a purple sidebar listing all sections as plain links. New: collapsible dark sidebar with hierarchical chapter/section structure and active section highlighting.
Typography Old: browser defaults, no visual hierarchy. New: carefully chosen font sizes, weights, and spacing. Entry numbers in accent blue. Clear h1/h2/h3 hierarchy.
Code Blocks Old: white background, browser monospace, no distinction from surrounding text. New: dark theme (Catppuccin-inspired), rounded corners, proper padding and overflow handling.
Tip Boxes Old: plain text, indistinguishable from regular content. New: light blue background with accent border, visually distinct callout style.

A Single Prompt, A Complete Redesign

What makes this interesting isn't just the visual improvement — it's how little guidance was needed. I didn't provide wireframes, color palettes, or typography specs. The prompt was essentially: here's the old site, make it modern. Claude analyzed the content structure (chapters, sections, code listings, tips, tables) and made its own design decisions. The result is a self-contained single HTML file with embedded CSS and JavaScript — no build tools, no dependencies, no frameworks.

Key numbers: 9 chapters, 68 sections, 292 code listings, 74 tip boxes, over 10,000 lines of HTML — all in one self-contained file per language.

Two Languages, Same Structure

After the redesign, Claude translated the entire book from German to English. The result: two files with identical DOM structure, identical code blocks, and matching anchor links. Every section number, every listing, every tip box exists in both versions at the same structural position.

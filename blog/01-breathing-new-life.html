Breathing New Life into [CDI@Work]:AI

February 2026

Somewhere on my hard drive sat a book I wrote years ago — [CDI@Work]:AI, a deep dive into Contexts and Dependency Injection for Java EE 6, featuring Apache DeltaSpike. It was written in German, originally authored in LaTeX, and at some point generated into a basic website. Over time it simply… aged. The specifications moved on. The ecosystem evolved. The book stayed frozen.

Then I had an idea: what if AI could help bring it back?

Step 1 — A Modern Web Version

The old LaTeX-generated website served its purpose, but it looked dated. I handed it to Claude with a fairly general prompt — turn this into a modern website. No detailed specs, no wireframes, no specific requirements for layout or typography. Claude analyzed the content, made its own design decisions, and produced a clean, responsive, single-file HTML book with a sidebar, embedded CSS and JavaScript, and all 278 code listings intact. It just figured out what a good version of this should look like.

Step 2 — The English Translation

With the new German edition in place, the next challenge was making the content accessible to a wider audience. I asked Claude to translate the entire book into English — but with strict rules: every code snippet, every CDI annotation, every DeltaSpike class name had to stay completely untouched. Only the surrounding German explanatory text should be translated, while the code examples remained in their original form.

What happened next was surprisingly smooth. Claude split the 10,000-line book into a dozen sections, translated them in parallel, and stitched everything back together. The result: two complete, self-contained HTML books with identical structure, identical code, and matching anchor links.

  • Transformed the old LaTeX-generated website into a modern, responsive HTML book
  • Analyzed the full book structure — 8 chapters, 278 code listings, 68 tips
  • Split into 12 parallel translation tasks for speed
  • Translated all German prose while preserving every code block byte-for-byte
  • Reassembled and verified structural integrity across both versions

What's Next

Here's the thing — the book covers CDI 1.0 through early CDI 2.0 concepts, all in the context of Java EE 6. A lot has changed since then. CDI is now at version 4.x, Jakarta EE has replaced Java EE, and DeltaSpike itself has continued to evolve.

The plan: With AI as a collaborator, the content of this book may receive a modernized update — refreshed for current specifications, with new examples and revised guidance. The original structure and teaching approach would stay, but the technical details would catch up with 2026. Stay tuned.

For now, both versions of the original book are available below. The German edition is the author's original text in a clean web format. The English edition is the AI-translated counterpart — same content, same code, different language.
